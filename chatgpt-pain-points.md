# Market Research: Customer Pain Points Analysis

**Platform:** ChatGPT (GPT-4 / GPT-5)

---

## System Message

You are an expert product strategist and market researcher specializing in customer pain point analysis. Your expertise includes:
- Conducting qualitative and quantitative research
- Identifying patterns in customer feedback
- Prioritizing problems based on impact and frequency
- Translating customer insights into product opportunities
- Using frameworks like Jobs to Be Done and Value Proposition Canvas

Provide thorough, data-driven analysis with actionable recommendations.

---

## User Message

Help me conduct a comprehensive customer pain point analysis for my product.

### Product Context
- **Product:** [Name and brief description]
- **Target Market:** [Industry, company size, roles]
- **Stage:** [Startup / Growth / Established]
- **Competition:** [Main alternatives customers consider]

### Available Research Data
- Customer interviews: [Count and segments]
- Support ticket database: [Timeframe and volume]
- Sales call recordings: [Number available]
- User behavior analytics: [Key metrics tracked]
- Customer surveys: [Response count]
- Online reviews/social mentions: [Sources]

### Analysis Request

#### 1. Pain Point Identification

For each pain point discovered, provide:

| Element | Description |
|---------|-------------|
| **Pain Point Name** | Clear, customer-centric label |
| **Problem Description** | In customer's language, 2-3 sentences |
| **Severity** | 1-10 scale with justification |
| **Frequency** | 1-10 scale with justification |
| **Business Impact** | Quantified where possible (time, cost, revenue) |
| **Emotional Impact** | Frustration, anxiety, stress levels |
| **Current Workarounds** | How customers cope today |
| **Supporting Evidence** | Quotes, data points, examples |
| **Affected Segments** | Which customer types experience this |
| **Root Cause** | Underlying reason for pain |

#### 2. Prioritization Framework

Create a prioritization matrix using:
- **Formula:** Priority Score = (Severity × Frequency × Market Size) / Current Solution Quality
- **Segments:** Categorize into Critical, High, Medium, Low priority
- **Rationale:** Explain ranking methodology

#### 3. Strategic Analysis

Provide insights on:

**Pattern Recognition:**
- Common themes across multiple pain points
- Related or cascading pain points
- Systemic vs. isolated problems

**Segment Analysis:**
- Pain points unique to specific customer types
- Universal pains vs. niche pains
- High-value segment priorities

**Competitive Landscape:**
- Pain points competitors solve well
- Gaps in current market solutions
- Emerging needs not yet addressed

**Opportunity Assessment:**
- Which pains align with our capabilities?
- Quick wins vs. strategic bets
- Differentiation opportunities

#### 4. Recommended Output Structure

Please format your response as:

**Executive Summary**
- 3-5 key findings
- Top 3 pain points to address
- Strategic recommendation

**Detailed Pain Point Catalog**
Table format with all identified pains ranked by priority

**Prioritization Matrix**
2×2 quadrant showing:
- Critical & Common (top priority)
- Painful but Rare (important edge cases)
- Frequent Annoyances (UX improvements)
- Low Priority (defer)

**Customer Segmentation View**
Pain points organized by customer type

**Strategic Recommendations**
- Immediate actions
- Near-term opportunities
- Long-term considerations
- Research gaps to fill

**Next Steps**
Suggested validation activities and timelines

---

### Additional Instructions

- Use tables and bullet points for scanability
- Include confidence levels for each insight (High/Medium/Low)
- Cite specific evidence where possible (e.g., "From interview #3: [quote]")
- Highlight any conflicting data or uncertainty
- Suggest metrics to track for each pain point
- Note any assumptions you're making about the data

### Example Output Snippet

**Pain Point:** "Manual data entry into multiple systems"
- **Severity:** 8/10 - Causes errors and delays critical workflows
- **Frequency:** 9/10 - Required multiple times daily
- **Impact:** 2-3 hours/day per user, ~30% error rate
- **Evidence:** "I spend half my morning just copying data between systems" (12/20 interviews)
- **Priority Score:** 8.5 (Critical & Common quadrant)
