# Buy vs Build Analysis for New Features
**Platform:** Claude (Sonnet 4.5 / Opus 4)

## Prompt
I need to decide whether to build a capability in-house or buy/partner for it. Help me conduct a rigorous buy-vs-build analysis.

**Feature/Capability:** [What you're considering]
**Strategic Importance:** [How critical to product]
**Timeline Pressure:** [How urgent]

## Analysis Framework

### Build Option Analysis
**Pros:**
- Full control and customization
- IP ownership
- Deep integration
- Long-term cost advantage
- Team learning and capability building

**Cons:**
- Higher upfront cost: [$X estimate]
- Longer time to market: [X months]
- Ongoing maintenance burden
- Technical risk
- Opportunity cost

**Requirements to Build:**
- Team: [X engineers for Y months]
- Skills needed: [Expertise required]
- Timeline: [Realistic schedule]
- Total cost: [Development + ongoing]

### Buy/Partner Option Analysis
**Pros:**
- Faster time to market: [X weeks/months]
- Lower upfront cost
- Proven solution
- Reduced technical risk
- Focus on core competencies

**Cons:**
- Vendor dependency
- Integration limitations
- Ongoing costs: [$X/month]
- Less differentiation
- Limited customization

**Available Vendors:**
| Vendor | Cost | Pros | Cons | Fit Score |
|--------|------|------|------|-----------|

### Decision Matrix
Score each option (1-10):
- Strategic fit
- Time to market
- Total cost (3-year)
- Technical risk
- Differentiation potential
- Team capability match

### Recommendation
Based on analysis, recommend build or buy with clear rationale.

Include:
- Decision criteria weighting
- Sensitivity analysis
- Hybrid options considered
- Timeline comparison
- Cost comparison (3-year TCO)
